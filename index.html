01 —

Executive Summary

ArtiFact addresses the most critical weakness in prior synthetic image detectors: poor generalisation to generators never seen during training. Earlier datasets like CIFAKE rely on a single generative model and low-resolution images — conditions that produce detectors brittle to any real-world deployment.

Published at IEEE ICIP 2023 by Rahman et al. from Bangladesh University of Engineering and Technology, ArtiFact assembles 2,496,738 images from 25 distinct generation methods — spanning 13 GANs, 7 diffusion models, and 5 other techniques — alongside 964,989 real images from 8 curated sources. All images undergo randomised JPEG compression and downscaling to simulate social media post-processing, the real-world condition that defeats most prior detectors.

The team's solution — a multi-class CNN with filter stride reduction — won 1st place in the IEEE VIP Cup at ICIP 2022, outperforming the second-best team by 8.34% on seen generators and a dramatic 15.08% on fully unseen generators.

Key Contribution

The fundamental innovation is a multi-class training strategy combined with filter stride reduction. Instead of training a binary real/fake classifier, each generator is treated as its own class — forcing the model to learn generator-specific artefact signatures. At test time these all collapse to a single "Fake" label. This approach dramatically improves generalisation to generators the model has never seen.

Dataset Composition

Unlike CIFAKE's balanced 50/50 split, ArtiFact reflects a realistic scenario: fake images outnumber real images by roughly 1.6:1, matching the asymmetric production capacity of generative models. The dataset is organised into per-generator subdirectories, each containing a metadata.csv with image path, binary target (0=real, 1=fake), and object category.

Data Generation & Pre-processing Pipeline

Every image in ArtiFact — whether real or synthetic — goes through a standardised transformation pipeline that applies realistic social media impairments. This is the factor that separates ArtiFact from earlier benchmarks: detectors trained here must be robust to compression artefacts and resolution degradation, not just raw pixel patterns from the generator.

05 —

Model Architecture

The authors' solution introduces two core innovations on top of a standard CNN backbone: a multi-class training objective and filter stride reduction. Together these address both the generalisation problem (unseen generators) and the social platform robustness problem (compression artefacts).

Core Innovation — Multi-Class Training

Instead of binary real/fake training, each generator is its own class. The model learns what makes a StyleGAN image different from a ProGAN image, what makes a DDPM image different from a Stable Diffusion image, and so on. This builds a rich internal representation of "what fakeness looks like" that generalises far better to unseen generators than a direct real/fake objective ever could.

Why Multi-Class Works — Explainability

The multi-class strategy works because different generative architectures leave fundamentally different artefact signatures. GANs produce checkerboard patterns from transposed convolutions and spectral anomalies at specific frequencies. Diffusion models produce different noise residuals and texture statistics. By learning to separate these, the model builds internal representations that transfer to generators it has never encountered.

GAN Artefact Signatures
Checkerboard patterns — transposed convolution artefacts appear in the frequency domain at regular intervals in many GAN architectures.
Spectral peaks — GAN-generated images show anomalous spikes in 2D Fourier spectra at frequencies corresponding to upsampling strides.
Texture inconsistency — backgrounds and fine-grained textures (hair, fur, fabric) often exhibit repetitive statistical patterns absent in real photography.
Diffusion Model Signatures
Denoising residuals — the iterative denoising process leaves subtle pixel-level noise patterns that differ statistically from camera sensor noise in real photographs.
Boundary artefacts — attention mechanisms in diffusion U-Nets create characteristic edge behaviour at object boundaries that real images do not exhibit.
Compression sensitivity — JPEG compression affects diffusion-generated images differently from real images at the same quality level, creating distinguishable DCT coefficient distributions.
Filter Stride Reduction — Why It Matters

Standard CNN architectures use stride-2 in early convolutional layers to quickly downsample spatial resolution. This discards the very high-frequency signals — above the Nyquist limit of the subsampled grid — that carry generator fingerprints. By reducing early-layer strides to 1 and delaying pooling, the model retains these signals long enough for subsequent layers to exploit them as features, directly explaining the large performance gap on unseen generators.

Using the Dataset

ArtiFact is organised into per-generator subdirectories. Each contains a metadata.csv with three columns: image_path, target (0=real, 1=fake), and category. Load all CSVs, aggregate, then use your standard image loader.

# Download via Kaggle API
kaggle datasets download -d awsaf49/artifact-dataset
unzip artifact-dataset.zip -d ./artifact
# PyTorch — Custom Dataset loading all metadata CSVs
import os, pandas as pd
from PIL import Image
from torch.utils.data import Dataset, DataLoader
import torchvision.transforms as T

# Aggregate all metadata.csv files
root = './artifact'
dfs  = []
for folder in os.listdir(root):
    csv = os.path.join(root, folder, 'metadata.csv')
    if os.path.exists(csv):
        df = pd.read_csv(csv)
        df['full_path'] = df['image_path'].apply(
            lambda p: os.path.join(root, folder, p))
        dfs.append(df)

meta = pd.concat(dfs, ignore_index=True)
# target: 0 = real, 1 = fake

class ArtiFact(Dataset):
    def __init__(self, df, transform=None):
        self.df = df.reset_index(drop=True)
        self.transform = transform

    def __len__(self): return len(self.df)

    def __getitem__(self, idx):
        row = self.df.iloc[idx]
        img = Image.open(row['full_path']).convert('RGB')
        if self.transform: img = self.transform(img)
        return img, int(row['target'])

transform = T.Compose([
    T.Resize((200, 200)),
    T.ToTensor(),
    T.Normalize(mean=[0.5,0.5,0.5], std=[0.5,0.5,0.5]),
])

loader = DataLoader(ArtiFact(meta, transform), batch_size=64,
                    shuffle=True, num_workers=4)
# TensorFlow / Keras
import tensorflow as tf, pandas as pd, os

root = './artifact'
paths, labels = [], []
for folder in os.listdir(root):
    csv = os.path.join(root, folder, 'metadata.csv')
    if os.path.exists(csv):
        df = pd.read_csv(csv)
        for _, row in df.iterrows():
            paths.append(os.path.join(root, folder, row['image_path']))
            labels.append(int(row['target']))

def load(path, label):
    img = tf.io.read_file(path)
    img = tf.image.decode_jpeg(img, channels=3)
    img = tf.image.resize(img, [200, 200])
    img = tf.cast(img, tf.float32) / 255.0
    return img, label

ds = tf.data.Dataset.from_tensor_slices((paths, labels))
ds = ds.map(load, num_parallel_calls=tf.data.AUTOTUNE)
ds = ds.shuffle(10000).batch(64).prefetch(tf.data.AUTOTUNE)
09 —

Applications & Limitations

ArtiFact's scale, generator diversity, and robustness to post-processing make it suitable for a range of high-stakes applications in media forensics and AI safety.

🌐 Social Media Moderation
The JPEG + downscale impairment pipeline directly targets the platform re-encoding conditions where most prior detectors fail. Models trained on ArtiFact are more deployment-ready than any single-generator baseline.
🔬 Generalisation Research
The three-tier test structure (seen / partial / unseen) provides a rigorous framework for evaluating cross-generator generalisation — a standard that other benchmarks lack entirely.
⚖️ Deepfake Detection
Human face and face-adjacent categories (FFHQ, AFHQ, CelebA-HQ, Metfaces) make ArtiFact directly applicable to deepfake detection pipelines for identity verification and media authentication.
📡 Frequency Domain Analysis
The diverse generator pool enables systematic study of artefact signatures in the spectral domain — identifying which frequency bands carry the most discriminative generator-specific information.
Key Limitations

Licence complexity — different generators carry different licences (some non-commercial only); commercial use requires per-source review. Class imbalance — 964K real vs 1.53M fake; use weighted loss or oversampling. Generator currency — the dataset covers models up to ~2022; SDXL, DALL-E 3, Midjourney v6, and Sora are not represented. Compute cost — 2.5M images at 200×200 requires substantial GPU resources and careful data pipeline engineering.
